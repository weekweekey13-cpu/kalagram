⚡ Калаграм не спит

Бесплатный Render засыпает ~через 15 минут без визитов. Внутренний таймер в приложении не помогает — он тоже спит. Нужен внешний пинг каждые 5 минут на URL ниже.

Пинг-URL (скопируй):
https://kalagram-z20h.onrender.com/api/ping Проверить сейчас

0. Cloudflare Worker — «второй сайт» который не спит

Бесплатный Worker на Cloudflare не засыпает как Render. Раз в 5 минут он пингует Калаграм. Код лежит в репозитории: scripts/cloudflare-ping-worker.js

  1. dash.cloudflare.com → регистрация
  2. Workers & Pages → Create → Create Worker
  3. Вставь код из cloudflare-ping-worker.js → Deploy
  4. Triggers → Cron Triggers → */5 * * * * (каждые 5 мин)
Открыть Cloudflare

1. UptimeRobot — самый простой (тоже отлично)

Бесплатно, пинг раз в 5 минут, почти никогда не опаздывает.

  1. Открой uptimerobot.com → регистрация
  2. Add New Monitor
  3. Monitor Type: HTTP(s)
  4. Friendly Name: Kalagram
  5. URL: вставь URL выше (/api/ping)
  6. Monitoring Interval: Every 5 minutes
  7. Create Monitor
Открыть UptimeRobot

2. cron-job.org — запасной вариант

  1. Регистрация
  2. Create cronjob → URL = пинг выше
  3. Schedule: every 5 minutes
  4. Enable → Create
Открыть cron-job.org

3. GitHub Actions (если репозиторий подключён)

В репозитории есть workflow Keep-alive каждые 5 минут. GitHub иногда задерживает cron на 5–15 мин — поэтому UptimeRobot важнее.

Проверка

После настройки открой Калаграм через 20–30 минут без визитов — должен открыться сразу, без минуты ожидания.
